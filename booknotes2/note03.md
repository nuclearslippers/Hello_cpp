# 数据介绍

本章主要介绍C++中的各种数据类型以及数据之间的转化方法。
```面向对象编程（OOP）的本质是设计并扩展自己的数据类型。```

## 变量的命名规则
1. `__name`和`_Name`是编译器使用的，不要使用
2. `_name`只用作局部变量的名字，因为编译器会使用它作为全局变量。

## 整型
C++的基本整型（按宽度递增的顺序排列）分别是 char、short、int、long 和 C++11 新增的 long long，其中每种类型都有符号版本和无符号版本。

### 1. short、int、long 和 long long
C++提供了一种灵活的标准，它确保了最小长度
1. short 至少 16 位；
2. int 至少与 short 一样长；
3. long 至少 32 位，且至少与 int 一样长；
4. long long 至少 64 位，且至少与 long 一样长。

具体到我们熟悉的操作系统中
| 操作系统 | short | int | long | long long |
| --- | --- | --- | --- | --- |
| Windows 64 位 | 16 | 32 | 32 | 64 |
| Linux 64 位 | 16 | 32 | 64 | 64 |

Q：我们该如何选择？
A：默认使用 int，当知道要超过 16 位时，使用 long。超过 20 亿则使用 long long。


### 无符号类型
`unsigned int`通过增加`unsigned`修饰符来创建无符号类型。

_关于溢出问题：C++ 标准明确规定了无符号类型的溢出行为，会自动进行模运算（即 “环绕” 行为）。但没有规定有符号整数溢出的行为，这种情况被称为未定义行为（Undefined Behavior, UB）。实际中，大多数编译器（如 GCC、Clang、MSVC）在默认设置下会表现出 “环绕” 行为（类似无符号类型），但这不是标准要求的。_

### 字符char
顾名思义，char 类型是专为存储字符（如字母和数字）而设计的。
C++对字符用单引号，对字符串使用双引号。

C++11 新增的类型：char16_t 和 char32_t。是无符号的，前者长 16 位，而后者长 32 位。
C++11 使用前缀 u 表示
char16_t 字符常量和字符串常量；并使用前缀 U 表示 char32_t 常量

## 常量
在C中，可以使用预处理器方法`#define`来定义常量。C++有一种更好的处理符号常量的方法：使用 `const` 键字来修改变量声明和初始化。

`const` 比 `#define` 好。首先，它能够明确指定类型。其次，可以使用 C++的作用域规则将定义限制在特定的函数或文件中。第三，可以将 `const` 用于更复杂的类型

## 浮点数

计算机将这样的值分成两部分存储。一部分表示值，另一部分用于对值进行放大或缩小。类似与科学计数法，不过计算机内部是二进制。

1. 表示方法
直接表示法：`11.22`
E表示法(也可以用e)：`1.122e1`

2. 浮点类型
`float` `double` `long double`
具体的位数跟平台和编译器有关。大致可以记一个：`float` 4字节，`double` 8字节，`long double` 16字节（windows可能还是8字节）。

浮点常量在默认情况下为 double类型，例如`1.0`为double类型。
